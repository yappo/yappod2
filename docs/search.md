# 検索

## 語彙検索

題名、本文、パッセージをトークンへ分割し、BM25Fで順位を計算します。既定の係数は`k1 = 1.2`、`b = 0.75`です。
複数セグメントの統計、新しい更新の優先、削除記録、メタデータフィルターを同じスナップショットで評価します。
AND、OR、フレーズ検索と、Block-Max WANDによる上位候補の取得に対応します。

## ベクトル検索

パッセージのベクトルは`vectors.yap2`へ保存し、`vectors.usearch`のHNSWから候補を取得します。公開するスコアは
元のベクトルで再計算します。ANN用コンポーネントがないセグメントでは全件を走査します。

- `cosine`はコサイン類似度です。ノルムが0のベクトルは0点です。
- `dot`は内積です。
- `l2`は負の二乗距離です。大きい値ほど近い結果です。

検索ベクトルの`dimensions`と有限値を検証します。`model_id`、`dimensions`、`metric`は索引作成時から固定されます。

## 組み合わせ検索

語彙検索とベクトル検索のスコアは尺度が異なるため、順位をReciprocal Rank Fusionで統合します。各候補の寄与は
`weight / (60 + rank)`です。片方だけに現れる候補も残し、両方に現れる候補は寄与を加算します。

## 文書とパッセージ

`documents`を指定すると、同じ文書に属する複数のパッセージを文書IDへ集約します。`passages`では該当する
パッセージをそのまま返します。すべての方式で同じgenerationのスナップショット、フィルター、新しい更新を優先する規則を使います。

## metadataによる絞り込み

`metadata.filterable_fields`へ登録したJSONパスだけを検索時に評価できます。HTTP APIの`filter`は一致、集合、
範囲、存在確認、論理演算を扱います。索引に登録していないフィールドを後から検索対象にはできません。

## RAG向けのパッセージ取得

`/v2/retrieve`は検索結果から引用可能な`context`を組み立てます。パッセージ本文、文書ID、URL、題名、元文書内の
文字位置、`context`内のバイト位置、各スコアを返します。回答生成は行いません。

全体のパッセージ数、1文書あたりの件数、`context`のバイト数を上限として評価します。収まらないパッセージは本文を途中で
切らずに飛ばします。別generationの検索結果やIDと`ordinal`の不一致は、安全側に倒して拒否します。
