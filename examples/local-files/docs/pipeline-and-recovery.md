# local-filesの処理工程と再生成

## 成果物

各処理段階は複数のNDJSON分割ファイル、`manifest.json`、設定の指紋値、チェックサムを保存します。`all`は完成済みの
処理段階について
次を確認してから再利用します。

- 入力ファイルのスナップショットが一致します。
- 前段階のmanifestとチェックサムが一致します。
- 一連の処理に関する設定の指紋値が一致します。
- チェックポイント、ジャーナル、公開済みの分割ファイルが矛盾していません。

チェックポイントだけを見て再開せず、入力とmanifestを含む全条件を確認します。

## 索引作成

複数の文書分割ファイルはコマンド内部のFIFOから1回の`yappo_makeindex build`へ渡します。シェルで巨大な中間ファイルへ
連結する必要はありません。索引作成が成功するまで最終索引ディレクトリは公開しません。

## ファイルごとの失敗

抽出できないファイルは失敗記録用の分割ファイルへパス、コード、メッセージ、抽出処理名を記録します。正常なファイルが
1件以上あれば文書分割ファイルを公開します。設定不正、manifest不一致、embedding失敗、索引作成の失敗は
コマンド全体を失敗させます。

## 全再生成

入力や設定が変わり、既存の処理段階を検証できない場合は、個別ディレクトリだけを削除しません。必要な証跡を保存し、
`examples/data/local-files`に再生成できないファイルがないことを確認してから全体を削除します。

```sh
rm -rf examples/data/local-files
```

その後、同じ設定で`all`を再実行します。削除対象には文書、パッセージ、ベクトル、索引、manifest、チェックポイント、
ジャーナルが含まれます。元の入力ファイルは削除しません。
