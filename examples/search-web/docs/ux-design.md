# Yappod2検索Web UIのUX設計

この文書では、利用者が検索、原文確認、質問、文書登録を迷わず完了できる画面構成を説明します。

## 利用者が行う操作

1. 調べたい語句を入力して検索します。
2. 題名、出典、本文の抜粋を比較します。
3. URLがある文書は参照先を開き、local-files文書では相対パスを確認します。
4. 必要に応じて文書を登録し、登録結果を検索します。
5. 質問への回答と、その根拠になった本文を同じ画面で確認します。

スコア、generation、文書IDは診断には必要ですが、通常の結果比較では補助情報です。そのため、題名、出典、
本文の抜粋を先に表示し、診断情報は展開できる詳細領域へ置きます。

## 画面構成

- 共通ヘッダーには、サービス名、検索・質問・文書登録の切り替え、デーモンの接続状態を置きます。
- 検索画面には、検索欄、実行ボタン、件数、結果一覧、続きを読み込むボタンを順に置きます。
- 質問画面には、質問欄、回答、参照資料を読む順に置きます。会話履歴を中心にした画面にはしません。
- 文書登録画面は検索から分離し、管理操作が必要な場合だけ開けるようにします。

画面幅が広い場合も本文を読みやすい幅の1列にします。狭い画面では入力欄と主要ボタンを縦に並べますが、
情報の順序は変えません。

## 状態と復旧

| 状態 | 表示 | 次にできること |
|---|---|---|
| 初期状態 | 入力例と短い説明を表示します。 | 検索語を入力できます。 |
| 処理中 | 操作位置の近くに進行中であることを表示します。 | 入力を保持したまま完了を待てます。 |
| 0件 | 一致する文書がないことを表示します。 | 語句を短くするか、表記を変えられます。 |
| 入力不正 | 対象フィールドの近くに理由を表示します。 | 入力を直して再実行できます。 |
| デーモン停止 | 共通ヘッダーと本文に接続不能を表示します。 | デーモン起動後に再接続できます。 |
| 検索失敗 | 検索条件を残してエラーを表示します。 | 同じ条件で再実行できます。 |
| LLM未設定 | 参照資料だけを利用できることを表示します。 | 本文と出典を確認できます。 |
| LLM失敗 | 取得済みの参照資料を残します。 | 再実行するか、資料だけを確認できます。 |
| 引用不正 | 回答を隠し、対応しない番号があることを表示します。 | 取得済み資料を確認できます。 |

## 視覚と操作

本文背景は白、文字は濃い灰色、リンクと主要操作には抑えた青緑を使います。情報の階層は文字サイズ、太さ、余白、
罫線で表します。装飾だけの巨大な見出し、過剰なカード、丸いラベル、グラデーション、ガラス風の表現、会話の吹き出し、
意味のないダッシュボード用サイドバーは使いません。

キーボードフォーカスを見える状態にし、主要操作領域は44px以上、本文は16px以上にします。状態の違いを色だけで
表現しません。URLのないlocal-files文書を誤ってリンクにせず、回答の引用番号から対応する資料へ1操作で移動できる
ようにします。
